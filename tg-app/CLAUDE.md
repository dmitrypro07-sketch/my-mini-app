# CLAUDE.md — навигация по проекту tg-app/

Telegram Mini App — каталог AI-ассистентов для бизнеса.
Vanilla JS + HTML + CSS, без фреймворков.

---

## Структура файлов

```
tg-app/
├── index.html        ← точка входа, разметка всех экранов
├── CLAUDE.md         ← этот файл
├── css/
│   └── style.css     ← все стили (CSS-переменные, компоненты)
└── js/
    ├── data.js       ← данные: ассистенты, категории, CONTACT_URL
    └── app.js        ← вся логика: экраны, шторка, фильтры, Telegram API
```

## Где менять данные

### Добавить/изменить ассистента
→ `js/data.js`, массив `assistants`

Поля:
- `id` — уникальный номер (не повторять)
- `icon` — эмодзи
- `title` — название
- `desc` — краткое описание (2 строки на карточке)
- `tagline` — боль клиента (серый текст в шторке)
- `price` — цена (число, в рублях)
- `tag` — категория (должна быть в массиве `categories`)
- `badge` — `'Хит'`, `'Новинка'` или `''`
- `payUrl` — ссылка на оплату; `''` → кнопка будет "Написать нам"
- `proof` — социальное доказательство (строка или `''`)
- `features` — массив строк "Что вы получите"

### Вставить ссылку на менеджера / бота
→ `js/data.js`, строка 7:
```js
const CONTACT_URL = 'https://t.me/YOUR_USERNAME';
```

### Вставить ссылку оплаты для ассистента
→ `js/data.js`, поле `payUrl` у нужного ассистента:
```js
payUrl: 'https://yookassa.ru/...',
```
После этого кнопка автоматически сменится на "Купить за N ₽".

### Изменить категории-фильтры
→ `js/data.js`, массив `categories`

### Изменить ссылку на бота (экран Success)
→ `js/data.js`, строка 11:
```js
const BOT_URL = 'https://t.me/YOUR_BOT';
```

## Навигация между экранами

```
[Каталог] ──тап на карточку──► [Шторка]
                                    │
                    ┌───────────────┴───────────────┐
                закрыть / BackButton / свайп вниз   купить / написать нам
                    │                               │
               [Каталог]                     если payUrl → [Success]
                                             если нет → [чат с менеджером]

[Success] ──MainButton "Перейти к ассистенту"──► [бот в Telegram]
[Success] ──BackButton──► [Каталог]
```

## Экраны (index.html)

| ID | Описание |
|---|---|
| `screen-catalog` | Главный: шапка + фильтры + карточки |
| `screen-success` | После покупки: иконка ✅ + шаги + кнопка к боту |
| `sheet` | Шторка деталей (не экран, а overlay поверх catalog) |

Экраны переключаются через `showScreen('catalog')` или `showScreen('success')` в `app.js`.

## Telegram API — что используется

| Метод | Где |
|---|---|
| `tg.ready()` / `tg.expand()` | Инициализация в `app.js` |
| `tg.colorScheme` + `onEvent('themeChanged')` | Тёмная/светлая тема |
| `tg.BackButton` | Закрытие шторки / возврат на каталог |
| `tg.MainButton` | CTA в шторке и на Success screen |
| `tg.openLink(url)` | Открытие оплаты и контакта |
| `tg.HapticFeedback` | Тактильный отклик (тап карточки, смена фильтра) |

## Что НЕ реализовано в v1 (намеренно)

- Авторизация (Telegram даёт имя/ID через `initData` — форма не нужна)
- Корзина / несколько товаров одновременно
- Поиск по тексту (фильтров достаточно для 8 позиций)
- История покупок (нет бэкенда)
- Push-уведомления (реализуются через бота, не Mini App)
- Подтверждение оплаты от платёжного сервиса (нужен бэкенд/webhook)

## Тестирование

Открыть в браузере: `tg-app/index.html` — работает без Telegram.
Без Telegram SDK: `tg` будет `undefined`, приложение работает через фоллбэк (`window.open`).

Тестирование в Telegram:
1. Загрузить на хостинг (GitHub Pages / Vercel / Netlify)
2. В BotFather: `/newapp` → указать URL
3. Открыть через бота
