# Бриф разработки: Telegram Mini App — каталог AI-ассистентов

## Экраны и что на них

---

### Экран 1 — Каталог (главный экран)

**Что видит пользователь:**
```
┌─────────────────────────────┐
│  🤖 АИ-сотрудники           │  ← шапка (синий фон)
│  Готовые ассистенты         │
├─────────────────────────────┤
│  [ Все ] [Маркетинг] [Продажи] [Контент] [Авито]  │  ← фильтры-чипы
├─────────────────────────────┤
│ ┌─────────────────────────┐ │
│ │ 🏷️  Генератор офферов   │ │  ← карточка
│ │ Превращает размытое...  │ │
│ │ 990 ₽        Маркетинг │ │
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │
│ │ 📣  РОП            ХИТ  │ │  ← бейдж "Хит"
│ │ Анализирует диалоги...  │ │
│ │ 19 900 ₽       Продажи │ │
│ └─────────────────────────┘ │
│            ...              │
└─────────────────────────────┘
```

**Элементы:**
- Шапка: логотип + заголовок + подзаголовок (синий фон, белый текст)
- Фильтры-чипы: горизонтальная прокрутка, активный чип подсвечен акцентом
- Карточки: иконка-эмодзи, название, краткое описание, цена (акцент), тег категории
- Опционально: бейдж "Хит" или "Новинка" на отдельных карточках

**Действия:**
- Тап на чип → фильтрует каталог по категории
- Тап на карточку → открывает шторку с деталями (Экран 2)

---

### Экран 2 — Шторка ассистента (модальное окно)

**Что видит пользователь:**
```
┌─────────────────────────────┐
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│  ← затемнённый каталог
│                         [✕] │
│ ┌─────────────────────────┐ │
│ │ 🏷️                      │ │  ← иконка крупная
│ │ Генератор офферов        │ │  ← название
│ │ Продукт нормальный,      │ │  ← tagline (боль клиента)
│ │ а заявок мало?...        │ │
│ │                          │ │
│ │ ЧТО ВЫ ПОЛУЧИТЕ          │ │  ← заголовок секции
│ │ ✓ 3 оффера под разные... │ │
│ │ ✓ 10 заголовков для...   │ │
│ │ ✓ 5 сильных CTA          │ │
│ │ ...                      │ │
│ │                          │ │
│ │ 990 ₽                   │ │  ← цена крупно
│ │ [    Написать нам    ]   │ │  ← inline кнопка (фоллбэк)
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ [   Купить за 990 ₽   ]     │  ← tg.MainButton (нативная)
└─────────────────────────────┘
```

**Элементы:**
- Оверлей (затемнение, тап закрывает шторку)
- Кнопка ✕ (закрыть, правый верхний угол, 30×30px)
- Иконка-эмодзи (40px)
- Название ассистента (20px, жирный)
- Tagline — боль клиента курсивом или серым (14px)
- Блок "Что вы получите" — список с галочками ✓
- Цена (24px, акцентный цвет)
- Inline-кнопка "Написать нам" / "Купить" (фоллбэк для браузера)
- `tg.MainButton` — нативная кнопка Telegram внизу: "Купить за N ₽" или "Написать нам"
- `tg.BackButton` — стрелка назад в шапке Telegram

**Действия:**
- Тап ✕ или оверлей → закрыть шторку
- Свайп вниз (>80px от верха) → закрыть шторку
- Тап "Написать нам" / MainButton → открыть контакт менеджера (`tg.openLink`)
- Тап "Купить" / MainButton (когда есть payUrl) → открыть страницу оплаты (`tg.openLink`)
- Тап BackButton (← в шапке Telegram) → закрыть шторку

**Переходы:**
- Закрыть шторку → Экран 1 (Каталог)
- Нажать "Купить" → Экран 3 (Success) — после оплаты
- Нажать "Написать нам" → выход из Mini App в чат с менеджером

### Экран 3 — Success (после покупки)

**Что видит пользователь:**
```
┌─────────────────────────────┐
│                             │
│           ✅                │  ← большая иконка
│                             │
│   Оплата прошла!            │  ← заголовок
│                             │
│   Генератор офферов         │  ← что купил
│   активирован               │
│                             │
│   Что делать дальше:        │
│   1. Перейди в бота         │
│   2. Нажми /start           │
│   3. Выбери ассистента      │
│                             │
│ [  Перейти к ассистенту  ]  │  ← tg.MainButton
│                             │
└─────────────────────────────┘
```

**Элементы:**
- Иконка ✅ (большая, по центру)
- Заголовок "Оплата прошла!"
- Название купленного ассистента
- Инструкция "Что делать дальше" (3 шага)
- `tg.MainButton` — "Перейти к ассистенту" → ссылка на бота

**Действия:**
- Тап MainButton → открывает бота с ассистентом

## Чего НЕ будет в версии 1.0

| Что | Почему |
|---|---|
| Регистрация и авторизация | Telegram даёт имя/ID автоматически через `initData` |
| Корзина / несколько товаров | Каждый ассистент покупается отдельно |
| Поиск по тексту | Фильтров по категориям достаточно для 8 позиций |
| История покупок | Нет бэкенда в v1, храним в боте |
| Отзывы и рейтинги | Добавим в v2 как социальное доказательство |
| Административная панель | Данные ассистентов редактируются в `app.js` вручную |
| Push-уведомления | Реализуются через бота, не через Mini App |
| Личный кабинет | Нет необходимости в MVP |

## Технические требования

### Платформа
- Telegram Mini App (WebView)
- Только мобайл (телефон), десктоп-Telegram как бонус
- Без фреймворков: vanilla JS + HTML + CSS (быстрее грузится)
- Статика: нет сервера, нет БД в v1

### Дизайн
- Нативные цвета Telegram через CSS-переменные (светлая/тёмная тема)
- Минимальная высота кнопок: **44px**
- Отступы от краёв экрана: **16px**
- Нижний отступ под `tg.MainButton`: **80px**
- Шрифт: системный (`-apple-system, BlinkMacSystemFont, Segoe UI`)
- Анимации: не тяжелее 60fps (WebView на Android тормозит)

### Telegram API (обязательные вызовы)
```javascript
tg.ready()                          // инициализация
tg.expand()                         // полноэкранный режим
tg.MainButton.show() / hide()       // нативная CTA-кнопка
tg.BackButton.show() / hide()       // кнопка "назад"
tg.openLink(url)                    // открыть ссылку (оплата / чат)
tg.HapticFeedback.impactOccurred()  // тактильный отклик
tg.themeParams                      // цвета текущей темы
```

### Запрещённые практики
- ❌ `alert()` / `confirm()` → использовать `tg.showAlert()` / `tg.showConfirm()`
- ❌ `window.open()` для ссылок → только `tg.openLink()`
- ❌ Внешние CDN (jQuery, Google Fonts) → всё локально
- ❌ `localStorage` для чувствительных данных
- ❌ `100vh` без учёта клавиатуры (использовать `tg.viewportHeight`)
