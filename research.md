## 5 примеров Mini Apps в похожих нишах

---

### 1. Tribute (@tributeapp_bot) — витрина цифровых товаров

**Что делает:** Платформа монетизации для авторов — продажа подписок, цифровых продуктов, доступа к контенту прямо в Telegram.

**Экраны:** Витрина продуктов → Карточка товара → Оплата (Telegram Stars / карта) → История покупок

**Сильные стороны:**
- Очень быстрый flow покупки: 2–3 тапа от витрины до оплаты
- Нативная оплата через Telegram Stars — не нужно уходить из приложения
- Хорошо адаптирован под мобайл, крупные кнопки

**Слабые стороны:**
- Витрина у каждого автора отдельная — нет единого каталога
- Кастомизация витрины очень ограничена
- Нет поиска и фильтров

**Что мы можем сделать лучше:**
✓ Единый каталог всех ассистентов в одном месте
✓ Фильтрация по категориям (маркетинг, продажи, контент)
✓ Подробное описание каждого продукта с реальными примерами результата

### 2. Dikidi (@dikidi_bot) — запись к специалистам

**Что делает:** Запись в салоны красоты, барбершопы, к врачам. Полный цикл внутри Telegram.

**Экраны:** Каталог специалистов/услуг → Карточка мастера (фото, отзывы, цены) → Выбор времени → Подтверждение → Мои записи

**Сильные стороны:**
- Полный цикл записи без выхода из Telegram
- Уведомления о записи через Telegram
- Каталог с поиском, есть фильтры
- Карточка специалиста с отзывами — хорошо строит доверие

**Слабые стороны:**
- Медленная загрузка при слабом интернете
- Выбор слота на маленьких экранах неудобен

**Что мы можем сделать лучше:**
✓ Быстрая загрузка (статика, без тяжёлых зависимостей)
✓ Карточка ассистента с конкретными примерами результатов — аналог "отзывов"

### 3. Getgems (@getgems_bot) — NFT-маркетплейс

**Что делает:** Маркетплейс NFT на блокчейне TON. Один из самых технически сложных Mini Apps.

**Экраны:** Каталог коллекций → Фильтры → Карточка NFT с историей продаж → Профиль → Купить/Продать

**Сильные стороны:**
- Лучший каталог среди Telegram Mini Apps: хороший поиск и многоуровневая фильтрация
- Красивые карточки коллекций с превью изображений
- Грамотное использование пространства экрана

**Слабые стороны:**
- Сложен для аудитории без крипто-бэкграунда — нет онбординга
- Некоторые экраны перегружены информацией

**Что мы можем сделать лучше:**
✓ Простой онбординг — одна строка о том, что такое AI-ассистент
✓ Карточки без лишней информации — только самое важное

### 4. Fanton (@FantonBot) — маркетплейс карточек (фэнтези-футбол)

**Что делает:** Фэнтези-футбол с каталогом карточек игроков, маркетплейсом и турнирами.

**Экраны:** Каталог карточек с фильтрами → Карточка игрока → Маркетплейс (купить/продать) → Мои команды → Лидерборд

**Сильные стороны:**
- Быстрая загрузка карточек, плавные анимации
- Маркетплейс встроен нативно
- Правильно использует MainButton и BackButton от Telegram

**Слабые стороны:**
- Нет онбординга для новых пользователей
- Фильтры не всегда очевидны

**Что мы можем сделать лучше:**
✓ Тэги-категории видны сразу на карточке (как у нас уже есть)
✓ Понятные подписи к каждому ассистенту — без жаргона

### 5. Telegram Wallet (@wallet) — P2P маркетплейс крипты

**Что делает:** Официальный крипто-кошелёк Telegram с P2P-торговлей. Эталонный пример от самой Telegram.

**Экраны:** Витрина активов → P2P-список офферов с фильтрами → Карточка оффера → Оформление сделки → История

**Сильные стороны:**
- Эталонный UX — сделано командой Telegram
- Минималистичный UI, всё понятно без инструкций
- P2P-витрина: список с фильтрами по сумме и методу оплаты — отличный паттерн для каталога

**Слабые стороны:**
- Ограниченная география
- Верификация громоздкая

**Что мы можем сделать лучше:**
✓ Брать паттерн списка с фильтрами как основу навигации
✓ Минимализм в UI — не перегружать карточки

## Обязательные UI элементы

### Telegram-специфичные
- `Telegram.WebApp.MainButton` — главная кнопка CTA (купить, оформить)
- `Telegram.WebApp.BackButton` — кнопка "назад" при открытой карточке
- `Telegram.WebApp.themeParams` — цвета под тему пользователя (светлая/тёмная)
- `Telegram.WebApp.expand()` — развернуть на весь экран
- `Telegram.WebApp.initDataUnsafe` — имя пользователя без формы регистрации

### Общие
- Минимальная высота кнопки: **44px**
- Отступы от краёв: **16px**
- Нижний отступ под MainButton: **~80px**
- Индикаторы загрузки (skeleton / spinner)
- Пустые состояния (если нет результатов)

## Что мы можем сделать лучше конкурентов

1. **Конкретные примеры результатов** — в карточке показывать "до/после": что получит пользователь конкретными словами, а не абстрактными обещаниями
2. **Категории с фильтрами** — чипы "Маркетинг / Продажи / Контент / Авито" сверху
3. **Быстрая загрузка** — мы уже статика, конкуренты часто тяжёлые React-приложения
4. **Простой flow покупки** — нажал "Купить" → сразу ссылка на оплату, без лишних шагов
5. **Нативные элементы Telegram** — MainButton и BackButton вместо кастомных кнопок
6. **Тёмная тема** — уже поддерживаем, многие конкуренты нет

## Экспертная оценка

---

### 👁️ UX-дизайнер

**Типичные ошибки новичков в похожих Mini Apps:**

- **Нет обратной связи на действие.** Нажал кнопку — ничего не изменилось визуально. Пользователь нажимает снова, думает что сломалось. У нас: кнопка "Купить" сразу должна давать реакцию — цвет, анимация, смена текста.
- **Модалка без жеста закрытия.** Пользователи привыкли свайпать вниз чтобы закрыть шторку. Если этого нет — раздражение. У нас шторка закрывается только по кнопке ✕ или оверлею — нужно добавить swipe-to-close.
- **Текст мелкий и серый.** Описания ассистентов в карточках часто читаются плохо на улице при ярком солнце. Минимум 14px, контраст достаточный.
- **Нет состояния загрузки.** Пользователь нажал "Купить" и ждёт — не знает, что происходит. Нужен хотя бы spinner или изменение кнопки.
- **Прокрутка шторки неочевидна.** Длинный список "Что вы получите" может обрезаться. Нужен визуальный намёк что там ещё есть контент ниже (градиент или частично видимый следующий элемент).

**Что обязательно должно быть:**
- Swipe-to-close для модальной шторки
- Визуальная реакция на каждый тап (active state)
- Иконки ассистентов крупные и понятные — сейчас эмодзи, это ок, но должны быть читаемы
- Цена выделена ярко — это главный триггер решения
- Кнопка "Купить" всегда видна без прокрутки

**Что бесит пользователей больше всего:**
1. Медленное открытие (> 2 сек) — бросают
2. Невозможность закрыть шторку свайпом
3. Мелкий текст описания
4. Кнопка "Купить" ведёт в никуда (наша текущая "Скоро") — лучше скрыть или заменить на "Написать нам"
5. Нет понимания "а что дальше после покупки?" — нужен Success screen

### 📈 Маркетолог

**Что заставит купить:**

- **Конкретный результат, а не процесс.** Не "анализирует диалоги", а "получите готовый ответ чтобы закрыть сделку прямо сейчас". В карточках уже есть это — хорошо, но можно усилить.
- **Социальное доказательство.** Даже одна цифра ("уже используют 200+ предпринимателей") или один отзыв резко поднимает конверсию. Без этого цифровой продукт воспринимается как кот в мешке.
- **Якорная цена.** РОП стоит 19 900 ₽ — если показывать его первым, остальные ассистенты за 990 ₽ кажутся дёшево и доступно. Порядок карточек влияет на восприятие цен.
- **Срочность или ограничение.** "Доступно 50 активаций" или "Цена до конца недели" — даже если условно, конверсия растёт. Без срочности откладывают на потом.
- **Бесплатный вход.** Один ассистент бесплатно или демо-версия — убирает барьер первой покупки. Попробовал → доверие → купил остальные.
- **Понятное "что дальше".** После покупки пользователь должен знать: куда идти, как запустить ассистента, что делать. Инструкция в Success screen = меньше вопросов в поддержку.

**Элементы повышающие конверсию в каталоге:**

| Элемент | Эффект |
|---|---|
| Тег "Хит" / "Новинка" на карточке | +15–25% кликов на карточку |
| Социальное доказательство в шторке | +20–30% к конверсии в покупку |
| Якорная дорогая позиция первой | Делает остальные "дешевле" |
| Кнопка "Попробовать бесплатно" | Убирает страх первой покупки |
| Конкретные цифры в описании ("за 15 минут", "3 оффера") | Повышает доверие |
| Ограничение ("осталось X мест") | Снижает откладывание |

**Что точно не работает:**
- Кнопка "Скоро" — убивает желание. Лучше "Написать и получить первым"
- Абстрактные обещания ("эффективно", "профессионально") без цифр
- Длинный список фич без акцента на главном результате

### ⚙️ Telegram-разработчик

**Ограничения платформы которые важно знать:**

- **Mini App открывается в WebView.** Это не браузер — некоторые CSS-свойства работают иначе, особенно `position: fixed` и `100vh` (высота учитывает клавиатуру не всегда корректно).
- **Нет доступа к файловой системе.** Всё только в памяти или через API бота. Для нашего каталога не критично, но история покупок без бэкенда не сохранится.
- **Telegram Stars vs внешняя оплата.** Stars — нативная оплата, одобряется без проблем. Внешние ссылки на оплату (ЮKassa, Stripe) — работают, но Telegram может показать предупреждение "уходите из приложения". Пользователи пугаются.
- **`initData` для авторизации.** Можно получить имя, username и ID пользователя без регистрации. Не нужны формы входа — это огромный плюс для конверсии.
- **Ограничение на внешние ресурсы.** Если грузить шрифты/иконки с Google Fonts или CDN — может тормозить или блокироваться. Лучше всё локально.

**Что работает хорошо:**

```javascript
// Нативная кнопка — самый надёжный CTA
tg.MainButton.setText('Купить за 990 ₽');
tg.MainButton.show();
tg.MainButton.onClick(() => { /* открыть ссылку */ });

// BackButton — пользователи ждут этого поведения
tg.BackButton.show();
tg.BackButton.onClick(() => closeModal());

// Тема — автоматически подхватывает светлую/тёмную
document.body.style.background = tg.themeParams.bg_color;

// HapticFeedback — тактильный отклик при нажатии
tg.HapticFeedback.impactOccurred('light');
```

**Что лучше не делать:**

- ❌ Не использовать `alert()` и `confirm()` — в WebView работают некорректно, используй `tg.showAlert()` и `tg.showConfirm()`
- ❌ Не делать анимации тяжелее 60fps — WebView на слабых Android-устройствах тормозит
- ❌ Не грузить внешние JS-библиотеки (jQuery, React) без необходимости — наш vanilla JS правильный выбор
- ❌ Не хранить чувствительные данные в `localStorage` — это не защищено
- ❌ Не открывать платёжные ссылки через `window.open()` — использовать `tg.openLink()` для корректного поведения

**Текущий статус нашего кода:**

| Аспект | Статус | Что сделать |
|---|---|---|
| `tg.ready()` и `tg.expand()` | ✅ Есть | — |
| Тёмная тема | ✅ Есть | Улучшить через `themeParams` |
| MainButton | ❌ Нет | Добавить в карточку |
| BackButton | ❌ Нет | Добавить при открытой шторке |
| HapticFeedback | ❌ Нет | Добавить на тап карточки |
| `tg.openLink()` для оплаты | ❌ Нет | Заменить `href` на `tg.openLink()` |
| Swipe-to-close шторки | ❌ Нет | Добавить touch события |
